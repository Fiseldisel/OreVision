# 🪨 OreVision — автоматическая классификация руд по шлифам

End-to-end система классификации руд по панорамным OM-изображениям полированных
шлифов. Заменяет ручную визуальную экспертизу воспроизводимым автоматическим
анализом: сорт руды, количественные доли, интерпретируемые карты, отчёты.

**Классы (экспертная логика ТЗ):**

| Сорт | Признак | Цвет на карте |
|---|---|---|
| Рядовая | преобладают обычные срастания сульфидов | 🟩 зелёный |
| Труднообогатимая | преобладают тонкие срастания (сульфиды замещены нерудной фазой) | 🟥 красный |
| Оталькованная | доля талька > 10% площади | 🟦 синий |

**Качество (валидация 220 изображений, сплит по пробам без утечки):**
macro-F1 = **0.913** (рядовая 0.926 / труднообогатимая 0.899 / оталькованная 0.913),
ROC-AUC (ovr) = **0.966**, accuracy 91.4%. Подробности: `models/metrics.json`,
`models/confusion_matrix.png`.

## Быстрый старт

### Веб-интерфейс (для геолога)

```bash
streamlit run app.py
```

- вкладка **Анализ** — загрузка изображений (или путь к файлу/папке на диске),
  карта классов поверх изображения, карта уверенности, таблица метрик,
  текстовое заключение, экспорт JPG/PDF/CSV;
- вкладка **Пакетная обработка** — папка целиком без участия пользователя;
- вкладка **О модели** — метрики валидации, матрица ошибок, легенда;
- порог талька и прозрачность маски — слайдеры в сайдбаре (вердикт панорамы
  пересчитывается мгновенно, без повторного инференса).

### Пакетная обработка (CLI, встраивание в лабораторный конвейер)

```bash
python -m orevision.cli --input "D:/шлифы/партия_07_2026" --out results/batch_07 --pdf
python -m orevision.cli --input "D:/шлифы/панорама.tif" --pdf          # один файл
python -m orevision.cli --input ... --no-overlay                        # только CSV, быстрее
```

Артефакты на каждый файл: `overlay.jpg`, `confidence.jpg`, `report.pdf`;
сводные: `summary.csv`, `run_params.json` (полный лог параметров для
воспроизводимости), `cli_log.txt`.

### Режим экспертной проверки (active learning)

Геолог исправляет модель прямо в интерфейсе, без ML-знаний:

1. **Спорные участки** — если модель ошиблась на участке панорамы, укажите
   верный класс → «В набор дообучения» (тайл сохраняется в нативном разрешении);
2. **Экспертная коррекция вердикта** — то же для целого снимка;
3. вкладка **🎓 Дообучение** — счётчики собранных примеров, журнал исправлений
   и две кнопки: «⚡ Быстрое дообучение» (~2 мин, тёплый старт от текущей
   модели) и «🔁 Полное переобучение» (~8 мин, с нуля). Прогресс обучения
   виден прямо в интерфейсе; предыдущая модель автоматически архивируется,
   и её можно вернуть кнопкой «↩ Откатить модель».

Примеры фидбэка хранятся в `data/feedback/<класс>/` с журналом происхождения
(`feedback_log.csv`: источник, координаты тайла, что предсказывала модель,
какой чекпойнт ошибался). В обучение они входят как отдельный источник —
всегда в train, никогда в валидацию, поэтому метрики остаются честными.

## Как это работает

1. **Тайловый анализ.** Панорамы (>30 Мпикс) режутся на тайлы 1024 px в нативном
   разрешении; обычные фото анализируются сеткой перекрывающихся кропов в
   масштабе обучения. Каждый тайл классифицирует **EfficientNetV2-S**,
   дообученная с ImageNet на 1100 размеченных фото.
2. **Агрегация.** Фото: усреднение вероятностей по кропам (TTA). Панорамы:
   доли площади по тайлам + **экспертное правило ТЗ** — тальк > 10% →
   оталькованная, иначе преобладание обычных/тонких срастаний.
3. **Интерпретируемость.** Карта классов (зелёный/красный/синий) и карта
   уверенности (viridis: жёлтое — уверенно, тёмное — спорно) поверх
   изображения; геолог видит, какие участки за какой вердикт «проголосовали».
4. **Устойчивость к условиям съёмки.** Датасет содержит два домена (ч1 —
   светлая тональность, ч2 — тёмная); обучение с агрессивными цветовыми
   аугментациями (jitter, grayscale, blur) заставляет модель опираться на
   морфологию срастаний. Валидация по доменам: ч1 F1≈0.87, ч2 F1≈0.92.
5. **Фильтр фона.** Почти однотонные тёмные тайлы (эпоксидка, края шлифа)
   исключаются из статистики.



## Производительность

| Сценарий | RTX 4070 | Оценка для T4 |
|---|---|---|
| Панорама ~150 Мпикс (~170 тайлов) | десятки секунд | ~1.5–3 мин ✅ (лимит 5 мин) |
| Обычное фото | ~1 с | ~2–4 с |
| Обучение (1100 фото, 32 эпохи) | 6.8 мин | ~25–40 мин |

Основное время на панораме — декодирование JPEG и чтение с диска, GPU-часть
занимает секунды. На CPU панорама обрабатывается за ~10–20 мин.

## Ограничения

- Доля талька оценивается **по тайлам** (шаг сетки 1024 px на панорамах) —
  это площадная оценка по классификации участков, а не попиксельная маска.
  Карта классов позволяет геологу визуально проверить каждый участок.
- Метрики масштаба (мкм/пиксель) в данных нет — доли считаются в % площади
  изображения; при появлении метаданных масштаба добавится пересчёт в мм².
