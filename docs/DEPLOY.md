## Вариант B: чистая машина / сервер жюри (T4)

```bash
git clone <репозиторий> orevision && cd orevision
python -m venv .venv && source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu126
pip install -r requirements.txt
# положить обученные веса models/best.pt (входят в поставку, ~80 МБ)
streamlit run app.py --server.port 8501
```

GPU не обязателен: на CPU обычное фото — секунды, панорама — минуты.
Доступ с другого компьютера: `streamlit run app.py --server.address 0.0.0.0`
и открыть `http://<ip-машины>:8501`.

## Чем проверить работу (файлы из датасета)

Корень данных: `C:\Users\user\Documents\my_project\data\Задача 3. Скажи мне, кто твой шлиф`

| Что проверяем | Файл | Ожидаемый вердикт |
|---|---|---|
| Панорама (тайловый режим, ~10 c на GPU) | `Панорамы\15.jpg` | Труднообогатимая |
| Панорама с тальком | `Панорамы\10.jpg` | Оталькованная (тальк ~90%) |
| Фото: рядовая | `Фото руд по сортам. ч1\Рядовые руды\2548036 5x 1.JPG` | Рядовая |
| Фото: тонкие срастания | `Фото руд по сортам. ч1\Труднообогатимые руды\2539439-3.JPG` | Труднообогатимая |
| Фото: оталькованная | `Фото руд по сортам. ч2\оталькованные\-13.jpg` | Оталькованная |

Сценарий демо в интерфейсе:

1. Вкладка **«Анализ изображений»** → вставьте путь к файлу в поле
   «…или путь к файлу/папке на диске» (или перетащите файл в загрузчик) →
   **«Анализировать»**.
2. Посмотрите вердикт, карту классов (вкладка «Карта классов»), карту
   уверенности, таблицу метрик; скачайте PDF-отчёт.
3. Подвиньте слайдер «Порог доли талька» — вердикт панорамы пересчитается
   мгновенно (без повторного инференса).
4. Вкладка **«Пакетная обработка»** → укажите папку `...\Панорамы` → на выходе
   `summary.csv` + PDF на каждый файл.

То же самое без интерфейса (CLI):

```powershell
C:\Users\user\anaconda3\python.exe -m orevision.cli --input "C:\...\Панорамы\15.jpg" --pdf
# результаты в results\run_<дата>\: overlay.jpg, confidence.jpg, report.pdf, summary.csv
```

## Деплой для жюри (публичная ссылка)

### Вариант 1: выданный сервер с T4 (рекомендуется)

У команды есть выделенный сервер с T4 (доступы — в письме от организаторов;
логин = имя zip-вложения, SSH порт 22). Машина чистая, стоит docker.

```bash
# 1) скопировать проект (данные НЕ нужны — только код, models/best.pt и demo/)
scp -r orevision <login>@<server_ip>:~/
ssh <login>@<server_ip>

# 2) окружение
cd ~/orevision
python3 -m venv .venv && source .venv/bin/activate
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu126
pip install -r requirements.txt

# 3) запуск на всех интерфейсах, переживает разрыв SSH
nohup streamlit run app.py --server.address 0.0.0.0 --server.port 8501 \
  > streamlit.log 2>&1 &
```

Ссылка для жюри: `http://<server_ip>:8501`. Проверить, что порт открыт:
`curl -I http://<server_ip>:8501` с локальной машины. Если снаружи закрыто
всё, кроме SSH — используйте ngrok (вариант 2) прямо на сервере.

> ⚠ VM удаляют в воскресенье 23:59 — если жюри проверяет позже, заранее
> написать организаторам о продлении, либо держать запасной вариант 2 локально.

### Вариант 2: публичная ссылка через ngrok (за 2 минуты)

Работает и на сервере, и на вашей машине (ngrok.exe уже есть локально):

```powershell
# терминал 1: приложение
cd C:\Users\user\Documents\my_project\orevision
C:\Users\user\anaconda3\python.exe -m streamlit run app.py --server.port 8501

# терминал 2: туннель (нужен бесплатный authtoken с ngrok.com)
C:\Users\user\Documents\yolo_test\ngrok.exe http 8501
```

ngrok выдаст `https://<случайное-имя>.ngrok-free.app` — эту ссылку и сдавать.
Машина должна быть включена, пока жюри проверяет.

### Что увидит жюри

Кнопки «Пример: рядовая / труднообогатимая / оталькованная / панорама» на
главной вкладке — демо в один клик без файлов; свои изображения жюри может
загрузить через браузер (до ~1 ГБ).

## Если что-то не работает

- «Нет чекпойнтов в models» — не хватает `models/best.pt`; обучить заново:
  `python -m orevision.tools.build_manifest --cache && python -m orevision.train`
  (пути к данным — в `config.yaml` → `data.root`).
- Порт занят — добавьте `--server.port 8502`.
- Изображение >1 ГБ через браузер — используйте поле «путь на диске», оно
  читает файл напрямую без загрузки.
